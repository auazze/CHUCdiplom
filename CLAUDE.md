# CHUCdiplom - Демонстрационный экзамен КОД 09.02.07-5-2026

## Что это

Комплект оценочной документации (КОД) для демонстрационного экзамена по специальности **09.02.07 "Информационные системы и программирование"**, квалификация "Специалист по информационным системам".

Утверждено ФГБОУ ДПО ИРПО, приказ от 29.09.2025 No 01-09-538/2025.

**Контекст:** Студент готовится к демонстрационному экзамену (через ~2 месяца). Экзамен будет **без интернета**, скорее всего на **C# (WinForms/WPF)** + **MS SQL Server**. Студент не знает C# и нуждается в обучении с нуля.

## Модули экзамена

### Модуль 1: Проектирование ER-диаграммы (все уровни)
**Задача:** Проанализировать исходные данные (JSON + Excel) и построить ER-диаграмму предметной области.

**Предметная область:** Молочный комбинат "Полесье" - производство молочной продукции.

**Исходные данные:**
- `Заказчики.json` - контрагенты (id, name, inn, addres, phone, salesman, buyer)
- `Цены.xlsx` - цены на продукцию и материалы
- `Заказ покупателя.xlsx` - документ заказа (шапка + табличная часть)
- `Спецификация.xlsx` - рецептура/BOM (из чего делать продукцию)
- `Производство.xlsx` - документ производства (выход + расход материалов)
- `Расчет стоимости продукции.xlsx` - калькуляция себестоимости

**Сущности ER-диаграммы:**
1. **Контрагент** (id PK, наименование, ИНН, адрес, телефон, поставщик, покупатель)
2. **Номенклатура** (код PK, наименование, ед_изм, тип [Продукция/Материал], цена)
3. **Спецификация** (id PK, наименование, код_продукции FK, кол-во, изготовитель_id FK)
4. **Строка_Спецификации** (id PK, спецификация_id FK, код_материала FK, количество)
5. **Заказ_Покупателя** (номер PK, дата, контрагент_id FK, исполнитель_id FK, сумма)
6. **Строка_Заказа** (id PK, заказ_номер FK, код_продукции FK, количество, цена, сумма)
7. **Производство** (номер PK, дата)
8. **Продукция_Производства** (id PK, производство_номер FK, код_продукции FK, количество)
9. **Материалы_Производства** (id PK, производство_номер FK, код_материала FK, количество)
10. **Расчет_Стоимости** (id PK, код_продукции FK, стоимость_итого)
11. **Строка_Расчета** (id PK, расчет_id FK, код_материала FK, количество, цена, стоимость)

**Ключевые связи:** Все 1:M. Контрагент -> Заказы, Номенклатура -> все документы, Документ -> Строки документа.

### Модуль 2: Разработка БД на основе ER-диаграммы (все уровни)
**Задача:** Создать базу данных в СУБД (MS SQL Server) по спроектированной ER-диаграмме. Импортировать данные из JSON-файла `Заказчики.json`.

**Что нужно уметь:** CREATE TABLE, PRIMARY KEY, FOREIGN KEY, типы данных, INSERT INTO.

### Модуль 3: Создание SQL-запроса (все уровни)
**Задача:** Написать SQL-запрос к созданной БД. Конкретный запрос будет в задании на экзамене.

**Что нужно уметь:** SELECT, JOIN, WHERE, GROUP BY, HAVING, ORDER BY, подзапросы, агрегатные функции.

### Модуль 4: Разработка информационной системы (БУ + ПУ)
**Время:** ~1 час
**Задача:** Разработать десктопное приложение (C# WinForms/WPF) с формами авторизации и администрирования.

**Форма 1 - Авторизация:**
- Поля: Логин, Пароль
- Кнопка "Войти"
- **КАПЧА-пазл:** 4 фрагмента изображения (аниме-персонаж), пользователь собирает картинку
- Валидация: непустые поля, проверка в БД
- При неверном вводе: сообщение об ошибке
- При успехе: сообщение об успехе + переход на соответствующую форму
- **После 3 неудачных попыток** (пароль ИЛИ капча) - блокировка аккаунта
- Заблокированный пользователь видит: "Вы заблокированы. Обратитесь к администратору"

**Форма 2 - Панель администратора (роль "Администратор"):**
- Список пользователей (DataGridView)
- Добавление пользователя (проверка на дубликат логина)
- Редактирование пользователя
- Разблокировка пользователей

**Форма 3 - Панель пользователя (роль "Пользователь"):**
- Базовая страница после авторизации

**Таблица Пользователи в БД:** ID, Login, Password, Role (Администратор/Пользователь), IsBlocked, FailedAttempts

**Требования к коду (из "Требования к разработке.pdf"):**
- Осмысленные имена переменных, методов, классов, контролов (не Form1, button3)
- Отдельные папки: Forms, Components, Entities
- Каждая сущность - отдельный класс
- Масштабируемый layout, минимальный размер окна
- Правильный Tab order
- MessageBox с правильным типом, заголовком и иконкой
- Обработка ошибок (приложение не должно крашиться)
- Один оператор на строку, комментарии только для неочевидного кода

### Модуль 5: Разработка проектной документации (БУ + ПУ)
**Задача:** Подготовить проектную документацию на разработанную систему.

### Модуль 6: Интеграция программных модулей (только ПУ)
**Время:** ~1 час
**Задача:** Работа с API эмулятора TransferSimulator + валидация данных + запись результатов в .docx.

**TransferSimulator:**
- HTTP-сервер на порту 4444 (Java/Ktor)
- Единственный endpoint: `GET http://localhost:4444/TransferSimulator/fullName`
- Ответ: `{"value": "Фамилия& Имя& Отчество"}` (с символом `&` как разделителем - это "запрещённый символ")

**Что нужно сделать:**
1. Заполнить шаблон `ТестКейс.docx` - тест-кейсы на проверку ФИО на запрещённые символы
2. Разработать приложение с GUI:
   - Кнопка "Получить данные" - GET-запрос к API, отображение ФИО
   - Кнопка "Отправить результат теста" - валидация ФИО + запись результата в .docx
3. Результат валидации ("Успешно"/"Не успешно") записывается в bookmarks в документе

**Что нужно уметь:** HttpClient, JSON-десериализация, работа с .docx (Open XML SDK или Interop), валидация строк.

## Быстрые шпаргалки для экзамена

### SQL: Создание таблицы
```sql
CREATE TABLE Контрагенты (
    Id INT PRIMARY KEY IDENTITY(1,1),
    Наименование NVARCHAR(255) NOT NULL,
    ИНН NVARCHAR(12),
    Адрес NVARCHAR(255),
    Телефон NVARCHAR(20),
    Поставщик BIT NOT NULL DEFAULT 0,
    Покупатель BIT NOT NULL DEFAULT 0
);
```

### C#: Подключение к БД
```csharp
string connectionString = @"Data Source=(localdb)\MSSQLLocalDB;Initial Catalog=DemExam;Integrated Security=True";
using (SqlConnection conn = new SqlConnection(connectionString))
{
    conn.Open();
    string query = "SELECT * FROM Users WHERE Login = @login AND Password = @password";
    using (SqlCommand cmd = new SqlCommand(query, conn))
    {
        cmd.Parameters.AddWithValue("@login", loginText);
        cmd.Parameters.AddWithValue("@password", passwordText);
        using (SqlDataReader reader = cmd.ExecuteReader())
        {
            if (reader.Read())
            {
                string role = reader["Role"].ToString();
            }
        }
    }
}
```
